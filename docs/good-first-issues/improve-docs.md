# Good First Issue: Improve Documentation

**Difficulty:** Easy
**Skills:** Technical writing, Markdown
**Files:** `docs/`, `README.md`, `CONTRIBUTING.md`

## Description

Clear documentation makes the project accessible to new users and
contributors. Improvements can range from fixing typos to adding new
sections.

## Ideas

- Fix typos, broken links, or outdated instructions in any `.md` file.
- Add a "FAQ" section to the README with common questions.
- Improve the architecture documentation in `docs/ARCHITECTURE.md` with
  diagrams or clearer explanations.
- Add code comments to under-documented source files (especially public
  headers in `include/shellkeep/`).
- Write a short tutorial for a specific workflow (e.g., "How to manage
  multiple servers with environments").

## Steps

1. Identify an area of the documentation that could be improved.
2. Make your changes. For Markdown, preview locally (most editors have a
   Markdown preview).
3. Verify that all instructions you write actually work by following them
   yourself.
4. Submit a PR titled "docs: <brief description of improvement>".

## Acceptance Criteria

- The documentation is factually correct.
- Instructions can be followed literally and produce the expected result.
- Language is clear, concise, and in English.
- No sensitive information is included.
